# Aim of the Practical

Design a distributed application using **MapReduce (Java)** to process a system log file and find users who have logged into the system for the maximum period using Hadoop in pseudo distributed mode.

---

# What Your Program Does

Input log file:

```txt
user1,120
user2,50
user1,30
user3,200
user2,100
user4,250
user3,20
```

Meaning:

| User  | Login Time |
| ----- | ---------- |
| user1 | 120        |
| user2 | 50         |
| user1 | 30         |
| user3 | 200        |

The program adds total login time for every user.

Final Output:

```txt
user1 150
user2 150
user3 220
user4 250
```

Maximum login time user:

```txt
user4 -> 250
```

---

# Complete Flow of Execution

```text
Input File
    ↓
Mapper
    ↓
Shuffle & Sort
    ↓
Reducer
    ↓
Final Output
```

# 6. What is HDFS?

HDFS is Hadoop’s storage system.

It stores files in distributed form across multiple machines.

---

# 7. Features of HDFS

1. Distributed storage
2. Fault tolerance
3. High throughput
4. Large file support
5. Replication support

---

# 8. HDFS Components

## A. NameNode

Master node of HDFS.

### Responsibilities

* Stores metadata
* Maintains file system tree
* Tracks blocks

---

## B. DataNode

Slave node.

### Responsibilities

* Stores actual data
* Sends heartbeat to NameNode
* Performs read/write operations

---

## C. Secondary NameNode

Takes checkpoints of NameNode metadata.

---

# 9. What is Block in Hadoop?

HDFS divides files into blocks.

Default block size:

* 128 MB (new Hadoop)
* 64 MB (old Hadoop)

---

# 10. What is Replication?

HDFS stores multiple copies of blocks.

Default replication factor:

```text
3
```

Benefits:

* Fault tolerance
* Data recovery

---

# 11. Hadoop Modes

| Mode               | Description            |
| ------------------ | ---------------------- |
| Standalone         | No daemons             |
| Pseudo Distributed | Single machine cluster |
| Fully Distributed  | Multiple machines      |

---

# 12. What is Pseudo Distributed Mode?

All Hadoop daemons run on one system but behave like a cluster.

Used for:

* Learning
* Testing
* Practicals

---

# 13. What is YARN?

Apache Hadoop YARN stands for:

```text
Yet Another Resource Negotiator
```

It manages resources and job scheduling.

---

# 14. Components of YARN

## A. ResourceManager

Master of YARN.

Responsibilities:

* Resource allocation
* Job scheduling

---

## B. NodeManager

Runs on every node.

Responsibilities:

* Manages containers
* Reports to ResourceManager

---

## C. ApplicationMaster

Manages execution of one application.

# 15. What is MapReduce?

MapReduce is a programming model for processing large data in parallel.

---

# 16. Phases of MapReduce

```text
Input
  ↓
Mapper
  ↓
Shuffle & Sort
  ↓
Reducer
  ↓
Output
```

---

# 17. What is Mapper?

Mapper reads input and converts it into key-value pairs.

Example:

Input:

```text
user1,120
```

Mapper Output:

```text
(user1,120)
```

---

# 18. What is Reducer?

Reducer processes grouped data and generates final output.

Example:

```text
user1 -> [120,30]
```

Output:

```text
user1 150
```

---

# 19. What is Shuffle and Sort?

Intermediate phase between Mapper and Reducer.

Functions:

1. Groups same keys
2. Sorts data
3. Sends grouped data to reducer

---

# 20. What is Combiner?

Mini reducer.

Used to reduce network traffic.

---

# 21. Advantages of Combiner

1. Faster processing
2. Less network usage
3. Improves performance

# 23. Why Hadoop Uses Writable Classes?

Because Hadoop uses serialization for data transfer.

Writable classes are optimized for distributed processing.

---

# 24. What is Serialization?

Conversion of object into byte stream.

Needed for:

* Network transfer
* Disk storage

---

# 25. What is JobTracker? (Old Hadoop)

Manages MapReduce jobs.

In Hadoop 2 it is replaced by YARN.

---

# 26. What is TaskTracker? (Old Hadoop)

Executes tasks on nodes.

Replaced by NodeManager.

---
